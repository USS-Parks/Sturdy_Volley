  `feedback_user_owns_session_cutoff`) — keep going prompt-by-prompt until the
  user says stop; never self-cut citing context budget.
- **Dev-scene wiring is four sites:** `registry.ts`, `dev-route.ts` `DEV_SCENES`,
  `TitleScene.ts` (menu item + `onSelect` case), `?scene=` route.
- **Run the FULL e2e suite** for any new scene / shared-rig / scene-lifecycle
  change (a new lab qualifies). The prod preview build strips `import.meta.env.DEV`,
  so dev-menu-only changes (e.g. the data report) are invisible to e2e — assert
  via `?scene=`/`?debug=` debug APIs instead.
- **rAF vs deterministic `tick()` race:** the scene render loop steps between
  `page.evaluate` calls, so transient per-frame state resets before a test reads
  it. Latch durable state (see NavLab's `lastRecovery`) for race-free assertions.
- **Patches share edges, never overlap** (NavLab slope bug) — overlap makes
  `patchAt` resolve to the first-listed patch.
- **Domestic animals use the grounded shared motor; wild fauna integrate a
  steering velocity** (flight/swim aren't grounded). The horse (044) is a ridden
  *grounded* body — use the motor.

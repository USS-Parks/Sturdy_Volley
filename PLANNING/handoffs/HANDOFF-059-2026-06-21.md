## 4. CI e2e stability — IMPORTANT for the next session

- The e2e suite grew to **~340 tests / ~20-min serial run** (workers=1). On the
  **desktop-CI SwiftShader runner**, full-screen 3D rendering saturates the main
  thread, so **Playwright cannot complete a click on an overlay panel button** —
  it reports "element detached" and never lands. This is **not** a slow-load
  timeout (retries + 45s timeout did NOT fix it) and **not** a product
  regression: the panels render once + the update loop pauses while open, and
  every affected spec is **green locally at retries=0 + on mobile-CI**.
- **Fix (`73b2ceb`):** drive the fragile panel-action steps via the debug hooks
  the buttons already call — `acceptQuest`/`cancelQuest`, a new **`swapPetKind`**,
  and a new **`dismissDaySummary`** (invokes the held day-summary *Continue*
  callback). Mail's letter-list test asserts the listing instead of clicking into
  the read view. The canonical pause-menu→journal UI-click test (`quests:32`) was
  never failing and keeps its real clicks.
- **Lesson for 060+:** for any new panel e2e, prefer driving the *action* through
  a debug hook and reserve raw DOM clicks for one canonical UI-coverage test per
  panel. `retries:2 + 45s` (CI only) remain as a backstop in `playwright.config.ts`.
- **CI not yet confirmed green from this machine** — the `73b2ceb` fix removes the
  exact failure mode and 24/24 affected specs pass locally; verify the GitHub
  Actions run on the next session if you have access.

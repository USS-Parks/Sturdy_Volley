## 3. Locked decisions / gotchas this session (do not re-trip)

- **Rich `festivalSchema` lives below `questRewardSchema`** in `schemas.ts` (it
  reuses it). Every gameplay field is **optional with a default** so a thin entry
  still parses — that's why Frostlight validates today.
- **Whole-day festival, window as flavor.** Festival-day behaviors (dressing,
  `byFestival` schedule layer, shop closure, the festival panel) key off "today is
  the festival day"; `startMinutes`/`endMinutes` drive only the "happening now" HUD
  flair. Keep it that way — the e2e is clock-independent because of it.
- **One-per-year reward gating** is keyed by `calendar.year` in `save.festivals`
  (`minigameWonYear` / `relationshipYear`). The minigame is a pure, seed-driven
  state machine (`startFestivalMinigame`/`tapFestivalSlot`) seeded from festival id
  + absolute day — the documented multiplayer hook.

  `projects`); real migration is Prompt 067.

  `BallastBayTownScene` does not host festivals yet (parity work, not 057 scope).
- **Shared reward summary** lives in `rewards.ts` (`describeReward` /
  `summarizeRewards`) — reuse it.
